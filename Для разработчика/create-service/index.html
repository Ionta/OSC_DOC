Создание сервиса

Сервисы, эта модули кода, к которым нету доступа по Http. Данные модули мы можем получать через DI.
Давайте рассмотрим создания такого модуля 

namespace ВашеПространствоИмён
{
    // Атрибут который указывает, что следующий класс является сервисом
    [Service(typeof(ITestService), ServiceType.Scoped)]
    public class TestService : ITestService
    {
        public string Exec(string text)
        {
            // тут может быть наша логика
            return text + text;
        } 
    }

    // Интерфейс нашего сервиса
    public interface ITestService
    {
        public string Exec(string text);
    }
}

Давайте разберём подробнее наш червис

// Атрибут который указывает, что следующий класс является сервисом
[Service(typeof(ITestService), ServiceType.Scoped)]

Здесь мы видим атрибут с следующими аргументами. 

typeof(ITestService) // этот аргумент указывает на тип интерфейса нашего сервиса
к сведению

атрибут typeof(ITestService) является не обязательным и вы его можете не указывать, тогда при получении сервисы вам необходимо указать тип сервиса. Указываете не ITestService service а TestService service.

Следующий аргумент это время жизни сервиса:
Transient: при каждом обращении к сервису создается новый объект сервиса. В течение одного запроса может быть несколько обращений к сервису, соответственно при каждом обращении будет создаваться новый объект.
Scoped: для каждого запроса создается свой объект сервиса.
Singleton: объект сервиса создается при первом обращении к нему, все последующие запросы используют один и тот же ранее созданный объект сервиса

Получение сервиса через DI

using Ionta.OSC.ToolKit.Controllers;

namespace ВашеПространствоИмён
{
    [Controller("путь_к_контроллеру")]
    public class Controller 
    {
        private readonly ITestService _service;
        // Получение сервиса через DI
        public Controller(ITestService service){
            _service = service;
        }
        [Post("путь_к_методу")] // пример обращения "/путь_к_контроллеру/путь_к_методу"
        public string Send(SendRequest request)
        {
            return _service.exec(request.Text);
        }
    }
    class SendRequest{
        public string Text{get; set;}
    }
}

Все сервисы мы можем получить через конструктор

Глобальные сервисы

Все сервисы которые мы создаём они доступны только внутри модуля, но мы также можем создать глобальные сервисы которые можно получить из других модулей.
Для этого нам нужно в сервисе указать атрибут [GlobalService]
Код нашего сервиса теперь следующий: 

namespace ВашеПространствоИмён
{
    // Атрибут который указывает, что сервис глобальный
    [GlobalService]
    // Атрибут который указывает, что следующий класс является сервисом
    [Service(typeof(ITestService), ServiceType.Scoped)]
    public class TestService : ITestService
    {
        public string Exec(string text)
        {
            // тут может быть наша логика
            return text + text;
        } 
    }

    // Интерфейс нашего сервиса
    public interface ITestService
    {
        public string Exec(string text);
    }
}

теперь наш сервис мы можем получить из любого модуля в системе

using Ionta.OSC.ToolKit.Controllers;

namespace ВашеПространствоИмён
{
    [Controller("путь_к_контроллеру")]
    public class Controller 
    {
        private readonly ServiceWrapper _service;
        // Получение сервиса через DI
        public Controller(IServiceProvider service){
            _service = provider.GetService("ITestService");
        }
        [Post("путь_к_методу")] // пример обращения "/путь_к_контроллеру/путь_к_методу"
        public string Send(SendRequest request)
        {
            return return await _service.Execute<string>("Exec", request.Text);
        }
    }
    class SendRequest{
        public string Text{get; set;}
    }
}

Чтобы вызвать метод сервиса мы используем _service.Execute<string>("Exec", request.Text)
В метод Execute передаётся Имя метода который мы хотим вызвать и его Аргументы
